## BUTTONS

Always prefer Bootstrap button classes as base.
Brand variants override Bootstrap's default blue:

  .btn-branded          → solid red button (primary action)
  .btn-outline-branded  → outlined red button (secondary action)
  .btn-primary          → Bootstrap blue (used for non-brand actions)

Do NOT use .btn-danger — use .btn-branded instead.
Disabled state preserves brand color with reduced interactivity.

---

## STATUS BADGES

Use class .item-status as the base wrapper.
Apply one of these modifier classes:

  .status-success    → green bg  / dark green text   (completed, paid)
  .status-warning    → yellow bg / dark yellow text  (pending, draft)
  .status-primary    → light blue bg / dark blue text (active, in progress)
  .status-danger     → pink bg / dark red text        (failed, overdue)
  .status-secondary  → gray bg / dark gray text       (inactive, unknown)
  .status-dark       → black bg / white text          (archived, special)

Font size: 0.7rem, font-weight: 500, border-radius: 12px, padding: 0.25rem 0.5rem

## CONVENTIONS & RULES

- Use Bootstrap utility classes first; only write custom CSS when Bootstrap cannot cover it
- Use CSS custom properties (var(--...)) for all brand colors — never hardcode hex values in components
- .po class = cursor: pointer (use on non-button clickable elements)
- .text-branded = brand red text color
- Toasts use .toast-cont with z-index: 100
- Language selector is a full-screen overlay modal, not a dropdown

---

## WHAT NOT TO DO

- Do not use .btn-danger for primary destructive actions — use .btn-branded
- Do not use inline font-size or color styles — use utility classes or CSS vars
- Do not add new color variables without updating this file
- Do not use Bootstrap's default blue primary for brand actions
